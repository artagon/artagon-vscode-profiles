## Contributing

Thanks for helping improve these VS Code profiles! This repo is structured for reproducibility and portability; please follow the steps below to keep things tidy.

### Prerequisites
- VS Code CLI on PATH (`code --version`)
- `jq` installed

### Workflow
1. Validate JSON after any edit:
   - `bash scripts/validate-json.sh`
2. Compose settings for affected profiles:
   - `bash scripts/compose-settings.sh [profiles...]`
3. Export profile bundles when changing settings or extensions:
   - `bash scripts/export-profiles.sh [profiles...]`
4. (Optional) Open profiles to verify in-editor:
   - `scripts/open-profiles.sh --skip-install [profiles...]`
   - `bash scripts/install-extensions.sh <profile>`
5. Update docs if behavior or profiles change (`README.md`, `agents/*`)

### Profiles & Overrides
- Shared editor defaults live under `_shared/` (crisp/retina).
- Per‑profile JSON overrides live under `_overrides/` (strict JSON). Use `"@extends"` to reuse fragments.
- Do not edit `_merged/` — it’s generated by the composer.

### Commit Hygiene
- Group related changes and keep messages clear.
- Don’t commit unrelated generated files.
- Consider running `scripts/tests/run.sh` before opening a PR.

### Opening a PR
- Use the PR template checklist.
- Include rationale and any screenshots/log output that clarify the change.
